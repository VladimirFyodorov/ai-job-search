# /add-portal

Adds a new job portal to hunter-v2's portal plugin system.

## Triggers
- `/add-portal <name> <url>` — e.g. `/add-portal hellowork https://www.hellowork.com`

## Invoke Sequence

### Step 0 — ACK and notify start

```python
from tools.notify import notify_start, notify_done, notify_error
react("👀")  # ACK first — before any work
notify_start("add-portal", sender=reply)
```

### Step 1 — Validate inputs
- `<name>`: lowercase, alphanumeric + hyphens only (regex `^[a-z][a-z0-9-]+$`); error if invalid
- `<url>`: must start with `https://`; error if missing or invalid

### Step 3 — Register in portals __init__.py
Append to tools/jobs/portals/__init__.py:
```python
from tools.jobs.portals.<name> import <Name>Portal
register(<Name>Portal())
```

### Step 4 — Confirm to user and notify done

```python
notify_done("add-portal", f"портал {name!r} создан", sender=reply)
```

Reply with:
- Portal file created: `tools/jobs/portals/<name>.py`
- Registered in portal registry as `"<Name>"`
- Next step: implement selectors in the `search()` method
- Verify with: `python -c "from tools.jobs.portals import list_portals; print(list_portals())"`

## Template variables
- `<name>`: lowercase slug (as passed to /add-portal)
- `<Name>`: title-cased version (e.g. "hellowork" → "Hellowork")
- `<url>`: the URL as passed

## Rules
1. Never make real network calls during skill execution — only write files
2. The generated portal stub must return `[]` on any error
3. The `source` field must be set to the portal name (title-cased)
4. Only generate one portal per invocation
